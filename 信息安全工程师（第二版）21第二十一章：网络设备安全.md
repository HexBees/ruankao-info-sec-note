# 信息安全工程师（第二版）章二十一：网络设备安全


## 一、网络设备安全概况

### 1-1 交换机安全威胁

历史演进：

  1. 第一代。集线器，工作在物理层（放大信号，延长传输距离）
  2. 第二代。以太网交换机，二层交换机，工作在数据链路层（MAC 识别）
  3. 第三代。三层交换机，工作在网络层（VLAN）
  4. 第四代。业务安全性、可靠性、QoS；增加防火墙、负载均衡、IPS 等功能
  5. 第五代。软件定义网络（SDN）

主要威胁：

  1. MAC 地址泛洪。攻击交换机 MAC 地址表
  2. ARP 欺骗。发送虚假 ARP 包更新交换机 ARP 缓存
  3. 口令威胁。
  4. 漏洞利用。

### 1-2 路由器安全威胁

  1. 漏洞利用
  2. 口令安全威胁
  3. 路由协议安全威胁
  4. DoS/DDoS 威胁
  5. 依赖性威胁（破坏路由器依赖的服务或环境）

## 二、网络设备安全机制与实现技术

### 2-1 认证机制

  * 本地管理：Console 口令、AUX 口令、VTY 口令、user 口令、privilege-level 口令
  * 网络管理：TACACS+ 认证、RADIUS 认证

### 2-2 访问控制

带外（out-of-band）访问：不依赖其他网络；带内（in-band）访问：提供网络支持

  1. CON 端口访问（默认设置访问，物理上访问设备）
  2. VTY 访问控制（终端模式，给设备配置固定IP地址通过网络访问，协议使用Telnet或SSH2，增加时间约束）
  3. HTTP 访问控制
  4. SNMP 访问控制（①SNMP 访问控制，提供社区字符串认证；②限制 SNMP 访问的 IP 地址；③关闭 SNMP 访问）
  5. 设置管理专网（方法：①将管理主机和路由器之间的全部通信进行加密，用 SSH 替换 Telnet；②在路由器设置包过滤规则，只允许管理主机远程访问路由器）
  6. 特权分级（权限分级机制，每种权限级别对应不同的操作能力）

### 2-3 信息加密

启用 service password-encryption

### 2-4 安全通信

2-4-1 SSH

  1. 使用 hostname 制定设备名称
  2. 使用 ip domain-name 配置设备域
  3. 使用 crypto key generate rsa 生成 RSA 加密密钥。建议最小密钥为 1024 位
  4. 使用 ip ssh 设置 SSH 访问
  5. 使用 transport input 命令配置使用 SSH

2-4-2 IPSec VPN

  1. 设置 ISAKMP 预共享密钥
  2. 创建可扩展的 ACL
  3. 创建 IPSec transforms
  4. 创建 crypto map
  5. 应用 crypto map 到路由接口

### 2-5 日志审计

控制台日志审计（Console logging）、缓冲区日志审计（Buffered logging）、终端审计（Terminal logging）、SNMP
traps、AAA 审计、Syslog

### 2-6 安全增强

  1. 关闭非安全的网络服务及功能
  2. 信息过滤
  3. 协议认证（OSPF 协议认证、RIP 协议认证、启动 IP Unicast Reverse-Path Verification）

### 2-6 物理安全

  1. 制定授权人安装、卸载和移动网络设备
  2. 制定授权人进行维护以及改变网络设备的物理配置
  3. 制定授权人进行网络设备的物理连接
  4. 制定授权人进行网络设备的控制台使用以及其他的直接访问端口连接
  5. 明确网络设备收到物理损坏时的恢复活成或者出现网络设备被篡改配置后的恢复过程

## 三、网络设备安全增强技术方法（下午题：※※※※※※※）

### 3-1 交换机安全增强技术方法

  1. 配置交换机访问口令和 ACL，限制安全登录（安全访问控制分两级：①第一级通过控制用户的连接实现；第二级：通过用户口令认证实现）
  2. 利用镜像技术监测网络流量
  3. MAC 地址控制技术（①设置最多可学习的 MAC 地址数，控制以太网交换机维护的MAC地址表的表象数量，默认没有限制；②设置系统 MAC 地址老化时间，默认300秒）
  4. 安全增强（①关闭不需要的网络服务；②创建本地账号；③启用SSH服务；④限制安全远程访问；⑤限制控制台的访问；⑥启动登录安全检查；⑦安全审计；⑧限制 SNMP 访问；⑨安全保存交换机 IOS 软件镜像文件；⑩关闭不必要的端口；关闭控制台及监测的审计；警示信息）

### 3-2 路由器安全增强技术

  1. 及时升级操作系统和打补丁
  2. 关闭不必要的网络服务（CDP、TCP、UDP Small、Finger、HTTP、BOOTP、IP Source Routing、ARP-Proxy、IP Directed Broadcast、IP Classless、ICMP、SNMP、WINS、DNS）
  3. 明确禁止不使用的端口
  4. 禁止 IP 直接广播和源路由
  5. 增强路由器 VTY 安全
  6. 阻断恶意数据包（①源地址生成来自内部网；②loopback 数据包；③ICMP 重定向包；④广播包；⑤源地址和目标地址相同）
  7. 路由器口令安全
  8. 传输加密
  9. 增强路由器 SNMP 的安全

## 四、网络设备常见漏洞与解决方法

### 4-1 网络设备常见漏洞

  1. 拒绝服务漏洞
  2. 跨站伪造请求 CSRF
  3. 格式化字符串漏洞
  4. XSS Cross-Site Scripting
  5. 旁路 Bypass something
  6. 代码执行 Code Execution
  7. 溢出 overflow
  8. 内存破坏 Memory Corruption

### 4-2 网络设备漏洞解决方法

  1. 及时 获取网络设备漏洞信息
  2. 网络设备漏洞扫描（端口扫描、通用漏扫、专用漏扫）
  3. 网络设备漏洞修补（修改配置文件、漏洞利用限制、服务替换、软件升级）
