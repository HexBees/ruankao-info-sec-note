

# 信息安全工程师（第二版）二十章：数据库系统安全

### 

## 一、数据库安全概况

概念：指数据库的 CIA 能够得到保障，涉及数据库管理安全、数据安全、数据库应用安全以及数据库运行安全

### 1-1 数据库安全威胁

  1. **授权的误用（Misuses of Authority）** （合法用户越权，授权用户不适当授予）
  2. **逻辑推断和汇聚（Logical Inference and Aggregation）** （运用逻辑将不太敏感的数据结合起来推断出敏感信息--数据汇聚安全问题）
  3. **伪装（Masquerade）（** 身份冒用）
  4. **旁路控制（Bypassing Controls）** （数据库后门，旁路绕过）
  5. **隐蔽信道（Covert Channels）** （利用非正常的通信途径传输数据以躲避数据库安全机制的控制，如共享内存、临时文件）
  6. **SQL 注入（SQL Injection）** （利用数据库应用程序的输入未进行安全检查的漏洞，欺骗数据库服务器执行恶意的 SQL 命令）
  7. **数据库口令密码破解** （撞库、字典攻击）
  8. **硬件及介质攻击** （物理攻击）

### 1-2 数据库安全隐患

  1. 数据库用户账号和密码隐患（密码合规性验证、默认账号、默认密码）
  2. 数据库系统扩展存储过程隐患
  3. 数据库系统软件和应用程序漏洞
  4. 数据库系统权限分配隐患
  5. 数据库系统用户安全意识薄弱
  6. 网络通信内容明文传递
  7. 数据库系统安全机制不健全

### 1-3 数据库安全需求

  1. 数据库标识与鉴别
  2. 数据库访问控制
  3. 数据库安全审计
  4. 数据库备份与恢复
  5. 数据库加密
  6. 资源限制（防止授权用户无限制的使用数据库服务器资源，CPU、缓存、介质等）
  7. 数据库安全加固
  8. 数据库安全管理

## 二、数据库安全机制与实现技术

### 2-1 数据库安全机制

  * **标识与鉴别** ：用户属性定义、用户-主体绑定、鉴别失败处理、秘密验证、鉴别时机、多重鉴别机制设置
  * **访问控制** ：会话建立控制、系统权限设置、数据资源访问权限设置
  * **安全审计** ：审计数据产生、用户身份关联、安全审计查阅、限制审计查阅、可选审计查阅、选择审计事件
  * **备份与恢复** ：备份和恢复策略设置、备份数据导入导出
  * **数据加密** ：加密算法参数设置、密钥生成与管理、数据库加解密操作
  * **资源限制** ：持久存储空间分配最高配额、临时存储空间分配最高配额、特定事物持续使用时间或未使用时间限制
  * **安全加固** ：漏洞修补、弱口令限制
  * **安全管理** ：安全角色配置、安全功能管理

### 2-2 数据库加密

加密类型：

  * 数据库网上传输的数据（SSL 协议）
  * 数据库存储的数据（库内加密、库外加密）

技术方法：

  * 基于**文件** 的数据库加密技术。（文件作为整体加密）
  * 基于**记录** 的数据库加密技术。（每一个记录加密成密文存储）
  * 基于**字段** 的数据库加密技术。（以不同记录的不同字段为基本加密单元进行加密）

### 2-3 数据库防火墙

主要作用：

  1. 屏蔽直接访问数据库的通道
  2. 增强认证
  3. 攻击检测
  4. 防止漏洞利用
  5. 防止内部高危操作
  6. 防止敏感数据泄露
  7. 数据库安全审计

### 2-4 数据库脱敏

匿名化手段：屏蔽、变形、替换、随机、加密

### 2-5 数据库漏洞扫描

概念：模拟黑客使用的漏洞发现技术，对目标数据库的安全性尝试进行安全探测分析，手机数据库漏洞的详细信息系，分析其不安全配置。

产品：NGSSQuirrel for Oracle、xSecure-DBScan

## 三、Oracle 数据库安全分析与防护

### 3-1 Oracle 数据库安全分析

  1. **用户认证** （一般认证：username+password，具有口令加密、账户锁定、口令生命期和过期、复杂度验证等功能；管理员认证：强认证、操作系统认证、口令文件认证；网络认证：第三方认证、PKI 认证、远程认证）
  2. **访问控制** （网络访问控制、数据对象授权控制）
  3. **保险库** （Database Vault，保护敏感数据，通过设置**安全域Realm** 和**命令规则 Command Rules** 对特权进行控制。具备防止数据系统未授权变更、多因素可信授权、职责隔离、最小化特权的功能）
  4. **安全审计和数据库防火墙** （可审计活动：登录尝试、数据库活动、对象存取；防火墙支持白名单、黑名单、基于例外名单策略）
  5. **高级安全功能** （透明数据加密 Transparent Data Encryption，数据屏蔽 Data Masking）

### 3-2 Oracle 安全最佳实践

  1. 增强 Oracle 数据库服务器的操作系统安全。（操作系统安全相关，如最小化安装、补丁安装、关闭不必要网络服务）
  2. 最小化安装 Oracle，删除不必要组件（减小攻击面）
  3. 安装最新的安全补丁
  4. 删除或修改默认的用户名和密码
  5. 启用认证机制
  6. 设置好的口令密码策略（登录次数、锁定时间、有效时间、复杂度检查）
  7. 设置最小化权限（UTL_FILE、UTL_HTTP、UTL_TCP、UTL_SMTP）
  8. 限制连接 Oracle 的 IP 地址
  9. 传输加密（TNS 协议）
  10. 启用 Oracle 审计（失败访问、安全事件日志）
  11. 定期查看 Oracle 漏洞发布信息，及时修补
  12. 实时 Oracle 灾备计划

## 四、MS SQL 数据库安全分析与防护

### 4-1 MS SQL 安全分析

  1. 用户身份认证（Windows 认证、混合认证）
  2. 访问控制（RBAC：固定服务器角色 Fixed Server Roles、固定数据库角色 Fixed Database Roles、应用角色 Application Roles）
  3. 数据库加密
  4. 备份、恢复机制（备份机制：静态备份、动态备份；备份方案：文件和文件组备份、事务日志备份、完全备份、差异备份；恢复机制：简单恢复、完全恢复、批量日志记录恢复）
  5. 安全审计

### 4-2 MS SQL Server 安全最佳实践

  1. 设置好的数据库密码安全策略
  2. 加强扩展存储过程管理，删除不必要的存储过程
  3. 网上数据加密传输
  4. 修改数据库默认的 TCP/IP 端口号
  5. 对 SQL 数据库访问的网络连接进行 IP 限制
  6. 启用 SQL Server 日志审计，记录所有的用户访问和分析安全事件日志
  7. 定期查看 MS SQL Server 漏洞发布信息，及时修补
  8. 保证 MS SQL Server 的操作系统安全性
  9. MS SQL Server 安全检测，制定容灾备份计划

## 五、MySQL 数据库安全分析与防护

### 5-1 MySQL 安全分析

  * 用户身份认证（用户名+口令）
  * 访问授权（五个授权表：user、db、host、tables_priv 和 column_priv）
  * 安全审计

### 5-2 MySQL 安全最佳实践

  1. MySQL 安装（建立单独启用的用户和组）
  2. 建立 MySQL Chrooting 运行环境（沙箱）
  3. 关闭 MySQL 远程连接
  4. 禁止 MySQL 导入本地文件
  5. 修改 MySQL 的 root 用户 ID 和密码
  6. 删除 MySQL 默认用户和 db
  7. 更改 MySQL 的 root 用户名
  8. 建立应用程序独立使用数据库和用户账号
  9. 安全监测
  10. 安全备份
