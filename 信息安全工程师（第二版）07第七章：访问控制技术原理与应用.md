

# 信息安全工程师（第二版）章七：访问控制技术原理与应用

## 一、访问控制概述

**访问控制概念** ：**对资源对象的访问者授权、控制的方法及运行机制** 。

  * 访问者：主体。可以是用户、进程、应用程序等
  * 资源对象：被访问对象。可以使文件、程序、数据等
  * 授权：访问者可以对资源对象进行访问的方式，如读、写、删除
  * 控制：对访问者使用方式的监测和限制，对是否许可用户访问资源对象作出决策。如允许、禁止等

**访问控制目标** ：

  1. 防止非法用户进入系统
  2. 组织合法系统对系统资源的非法使用

**依据** ：《信息安全技术 网络安全等级保护安全设计技术要求（GB/T 25070-2019）》

## 二、访问控制模型

xmlns='http://www.w3.org/2000/svg' width='1365' height='720'></svg>)参考模型

  1. **主体** 。操作实施人，通常是人、进程或设备。一般是代表用户执行操作的进程。
  2. **客体** 。被主体操作的对象。
  3. **参考监视器** 。**访问控制中决策单元与执行单元的集合体** 。控制主体对客体的操作，监督访问行为，记录安全事件至审计文件。
  4. **访问控制数据库** 。记录主体访问客体的权限及其访问方式的信息，提供访问控制决策判断的依据，也称**访问控制策略库** 。
  5. **审计库** 。存储主体访问客体的操作信息。

## 三、访问控制类型

### 3-1 自主访问控制

自主访问控制（Discretionary Access Control，DAC）指客体的**所有者按照自己的安全策略** 授予系统中的其他用户对其的访问权。

  * **基于行的自主访问控制**

在每个**主体** 上都附加一个该主题可访问的客体的明细表。

  1. **能力表（Capability List）** ：访问“钥匙”，决定用户能否对客体进行访问以及具有何种访问模式（读、写、执行）
  2. **前缀表（Profiles）** ：包括受保护客体名和主体对它的访问权限
  3. **口令（Password）** ：基于口令机制的自助存取控制机制中，每个客体都相应口令。

  * **基于列的自主访问控制**

在每个**客体** 上都附加一个可访问它的主体的明细表。

  1. **保护位（Protection Bits）** ：对主体、主体组以及客体拥有者指明一个访问模式集合，通常以比特位来表示访问权限。**UNIX/Linux采用**
  2. **访问控制表（Access Control List，ACL）** ：在每个客体上都附加一个主体明细表，表示**访问控制矩阵** 。表中每一项都是包括主体身份和主体对客体的访问权限。

  * **优点** ：访问机制简单、灵活
  * **缺点** ：依赖用户安全意识和技能，不能满足高等级安全要求

### 3-2 强制访问控制

概念：强制访问控制（Mandatory Access Control，MAC）指系统根据主体和客体的安全属性，以强制方式控制主体对客体的访问。（BiBa，BLP）

在**政府部门、军事和金融领域** ，常利用强制访问控制机制，将系统中的资源划分安全等级和不同类别，然后进行安全管理。

### 3-3 基于角色的访问控制（Role Based Access Control）

  * **基本要素** ：用户（U），角色（R），会话（S），权限（P）
  * 用户与角色是**多对多** 关系，角色与权限是**多对多** 关系
  * **系统管理员** 负责管理角色集合和访问权限集合，将权限赋予角色，把角色映射到用户
  * 特点：强大、灵活

### 3-4 基于属性的访问控制（Attribute Based Access Control）

概念：根据**主体的属性** 、**客体的属性** 、**环境的条件** 以及**访问策略** 对主体的请求操作进行授权许可或拒绝

## 四、访问控制策略设计与实现

### 4-1 访问控制策略

策略制定考虑方面：

  1. 不同网络应用的安全需求。防外还是防内
  2. 所有和应用相关的信息的确认。端口、IP
  3. 网络信息传播和授权策略。安全级别与分类
  4. 不同系统访问控制与信息分类策略的一致性
  5. 关于保护数据和服务的有关法规及合同义务
  6. 访问权限的更新与维护

**访问控制策略要素** ：控制对象、访问控制规则、用户权限、访问安全要求

**访问控制策略示例**
：机房访问控制策略、拨号访问控制策略、路由器访问控制策略、交换机访问控制策略、防火墙访问控制策略、主机访问控制策略、数据库访问控制策略、客户端访问控制策略、网络服务访问控制策略等

### 4-2 访问控制规则

  1. 基于用户身份的访问控制规则。（使用身份控制，如用户名+口令）
  2. 基于角色的访问控制规则。（分配）
  3. 基于地址的访问控制规则。（使用访问者所在的物理地址或逻辑地址空间控制，如IP、域名、物理地址）
  4. 基于时间的访问控制规则。（使用时间约束访问，如非工作时间禁止）
  5. 基于异常事件的访问控制规则。（使用异常事件触发控制，如登录失败触发冻结）
  6. 基于服务数量的访问控制。（使用系统所能承受服务数量实现控制，如抗DDoS）

## 五、访问控制过程与安全管理

### 5-1 访问控制过程

  1. 明确访问控制管理的资产
  2. 分析管理资产的安全需求
  3. 制定访问控制策略，确定访问控制规则以及用户权限分配
  4. 实现访问控制策略，建立身份认证系统，根据用户授权访问资产
  5. 运行与维护访问控制系统，及时调整访问策略

### 5-2 最小特权管理

  * **特权（Privilege）** ：是用户超越系统访问控制所拥有的权限。
  * **最小特权原则（Principle of Least Privilege）** ：系统中每一个主体只能拥有完成任务所**必要** 的权限集。
  * 最小特权管理的**目的** ：系统不应赋予特权拥有者完成任务的额外权限，组织特权滥用。

### 5-3 用户访问管理

用户登记、用户权限分配、访问记录、权限监测、权限取消、撤销用户

### 5-4 口令安全管理

  * 口令选择至少8个字符以上，包含大小写字母、数字、特殊字符
  * 口令与账号不能相同
  * 更换默认口令
  * 限制登录次数，一般为3次
  * 禁止共享账号和口令
  * 口令需加密存放，仅限超户读取
  * 禁止明文传输口令
  * 口令应有时效性
  * 对账号和口令进行爆破检测

## 六、访问控制主要产品与技术指标

### 6-1 访问控制主要产品

  1. **4A 系统** ：认证（Authentication）、授权（Authorization）、账号（Account）、审计（Audit），统一安全管理平台，平台**集中提供账号、认证、授权和审计等网络安全服务** 。常使用RBAC
  2. **安全网关** ：利用网络数据包信息和网络安全威胁特征库，对网络通信连接服务进行访问控制。如**防火墙，统一威胁管理等** 。
  3. **系统安全增强** ：利用强访问控制技术增强OS、DB系统的安全，防止特权滥用

### 6-2 访问控制主要技术指标

  1. 产品支持访问控制策略规则类型
  2. 产品支持访问控制规则最大数量
  3. 产品访问控制规则检查速度
  4. 产品自身安全和质量保障级别

## 七、访问控制技术应用

应用场景：

  1. 物理访问控制。（门禁、警卫、证件）
  2. 网络访问控制。（NAC、连接控制、区域划分）
  3. 操作系统访问控制。（文件、进程、内存访问控制）
  4. 数据库/数据访问控制。（DLP）
  5. 应用系统访问控制。
