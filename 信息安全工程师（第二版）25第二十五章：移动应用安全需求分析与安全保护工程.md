

# 信息安全工程师（第二版）章二十五：移动应用安全需求分析与安全保护工程

## 一、移动应用安全威胁与需求分析

基本组成：移动应用（App）、通信网络（无线网络、移动通信网络、互联网）、应用服务端（相关服务器，处理来自App的信息）

### 1-1 移动应用安全分析

  1. 移动操作系统平台威胁（iOS、Google）
  2. 无线网络攻击（通信内容窃听、假冒基站、网络域名欺诈、网络钓鱼）
  3. 恶意代码（流氓行为、资源消耗、恶意扣费、隐私窃取、远程控制、诱骗欺诈、系统破坏、恶意传播）
  4. 移动应用代码逆向工程（获取关键算法思路、窃取敏感数据） 
  5. 移动应用程序非法篡改

### 2-2 Android 系统安全机制

  1. **权限声明机制** （normal：不会带来实质伤害；dangerous：潜在威胁，如位置、消息；signature：统一签名的应用才能访问；signatureOrSystem：设备商使用）

  2. **应用程序签名机制** （APK 文件进行数字签名，安装的程序都必须拥有一个数字证书）

  3. 1**沙箱机制**（实现不同应用程序和进程之间的相互隔离 UserID）

     3.2**网络通信加密** （SSL/TSL）

  4. **内核安全机制** （分区、Linux ACL）

### 四、移动应用安全保护机制与技术方案

### 4-1 移动应用 App 安全加固

  1. 防反编译（程序文件进行加密、代码混淆：名字混淆、控制混淆、计算混淆）
  2. 防调试（设置调试检测功能，以触发反调试安全保护措施）
  3. 防篡改（数字签名、多重校验）
  4. 防窃取（加密）

### 4-2 移动应用 App 安全检测

检测内容：

  1. 身份认证机制检测
  2. 通信会话安全机制检测
  3. 敏感信息保护机制检测
  4. 日志安全策略检测
  5. 交易流程安全机制检测
  6. 服务端鉴权机制检测
  7. 访问控制机制检测
  8. 数据防篡改能力检测
  9. 防 SQL 注入能力检测
  10. 防钓鱼安全能力检测
  11. App 安全漏洞检测

检测工具：

  * 进程注入工具 Inject
  * HijackActivity 劫持检测工具
  * Jeb 静态逆向分析工具
  * APK 反编译工具 apktool
  * 抓包工具 Tcpdump/Wireshark
  * Android Hook 框架 Xposed
  * 分析工具 Burpsuite
  * 静态分析工具 Androguard
  * 逆向工具 Android Killer
