# 信息安全工程师（第二版）章十三：网络安全漏洞防护技术原理与应用


## 一、网络安全漏洞概述

  * **概念** ：漏洞-**脆弱性** 。一般指致使网络信息系统**安全策略相冲突的缺陷**
  * **影响** ：机密性受损、完整性破坏、可用性降低、抗抵赖性缺失、可控制性下降、真实性不保
  * **存在** ：硬件层、软硬协同层、系统层、网络层、数据层、应用层、业务层、人机交互层
  * **分类** ：**普通漏洞** （信息广泛公开，有修补方案）；**零日漏洞** （新发现，**无补丁** ）
  * **威胁** ：敏感信息泄露、非授权访问、身份假冒、拒绝服务
  * **漏洞治理** ：漏洞检测、漏洞修补、漏洞预防

## 二、网络安全漏洞分类与管理

### 2-1 网络安全漏洞来源

**非技术性安全漏洞** 主要来源：

  1. 网络安全责任主体不明确（部门、机构）
  2. 网络安全策略不完备
  3. 网络安全操作技能不足
  4. 网络安全监督缺失
  5. 网络安全特权控制不完备

**技术性安全漏洞** 主要来源：

  1. 设计错误（Design Error）
  2. 输入验证错误（Input Validation Error）
  3. 缓冲区溢出（Buffer Overflow）
  4. 意外情况处置错误（Exceptional Condition Handling Error）
  5. 访问验证错误（Access Validation Error）
  6. 配置错误（Configuration Error）
  7. 竞争条件（Race Condition）
  8. 环境错误（Condition Error）

### 2-2 网络安全漏洞分类

**2-2-1 CVE （Common Vulnerability and Exposure）漏洞分类**

  * **来历** ：美国 **MITRE** 公司建设和维护的**安全漏洞字典**
  * **内容** ：标识数字、安全漏洞简要描述、至少一个公开参考
  * **格式** ：CVE-Year-ID

**2-2-2 CVSS （Common Vulnerability Scoring System）**

  * **来历** ：**事件响应与安全组织论坛** （FIRST）发布的通用漏洞计分系统
  * **依据** ：**基本度量计分** （攻击向量、攻击复杂度、特权要求、用户交互、完整性影响、保密性影响、可用性影响、影响范文等参数）；**时序度量计分** （漏洞利用代码成熟度、修补级别、漏洞报告可信度等参数）；**环境度量计分（** 完整性要求、保密性要求、可用性要求、修订基本得分等参数）

**2-2-3 我国信息安全漏洞分类**

  * **国家信息安全漏洞库（CNNVD）** 漏洞分类分级标准
  * **国家信息安全漏洞共享平台（CNVD）** 漏洞分类分级标准（分类：11种；分级：高、中、低）

**2-2-4 OWASP TOP 10**

  * OWASP（Open Web Application Security Program）
  * 厂商自行推出漏洞分类规范（IBM、Symantec、Cisco、Oracle、Microsoft）

### 2-3 网络安全漏洞发布

  * **发布主体** ：软硬件开发商、安全组织、黑客、用户
  * **发布形式** ：网站、电子邮件、安全论坛
  * **发布内容** ：漏洞编号、发布日期、安全危害级别、漏洞名称、漏洞影响平台、漏洞解决建议

### 2-4 网络安全漏洞信息获取

  * CERT（Computer Emergency Response Team）（Carnegie Mellon）
  * Security Focus Vulnerability Database（Security Focus）
  * 国家信息安全漏洞库 CNNVD（中国信息安全测评中心）
  * 国家信息安全漏洞共享平台 CNVD（国家计算机网络应急技术处理协调中心）
  * 厂商漏洞信息

### 2-5 网络安全漏洞管理过程

  1. 网络信息系统资产确认
  2. 网络安全漏洞信息采集
  3. 网络安全漏洞评估
  4. 网络安全漏洞消除和控制
  5. 网络安全漏洞变化跟踪

## 三、网络安全漏洞扫描技术

  * **概念** ：网络安全漏洞扫描是一种用于检测系统中漏洞的技术，是具有漏洞扫描功能的软件或设备，也称漏洞扫描器
  * **功能模块** ：用户界面、扫描引擎、漏洞扫描结果分析、漏洞信息及配置参数库
  * **主机漏洞扫描器** ：无需建立网络连接，运行于目标系统之上，通过检查本地系统中关键性文件的内容及属性，发现漏洞。（COPS、Tiger、MBSA）
  * **网络漏洞扫描器** ：通过与待扫描目标主机建立网络连接后，发送特定网络请求进行漏洞检查。（检查能力受限于获取到的有限信息；Nmap、Nessus、X-scan）
  * **专用漏洞扫描器** ：针对特定系统的安全漏洞检查工具。（数据库漏洞扫描器、网络设备漏洞扫描器、Web漏洞扫描器、工控漏洞扫描器）

## 四、网络安全漏洞处置技术与应用

### 4-1 网络安全漏洞发现技术

  * **方法** ：人工安全性分析、工具自动化监测、人工智能辅助分析
  * **技术** ：文本搜索、词法分析、范围检查、状态机检查、错误注入、模糊测试、动态污点分析、形式化验证等
  * **工具** ：Flawfinder、Splint、W3AF、Wireshark、Metasploit、OllyDBG、Grep、MOPS

### 4-2 网络安全漏洞修补技术

  * 特点：系统的、周而复始的工作
  * 产品：SUS、PatchLink、LANDesk、VRVEDP

### 4-3 网络安全漏洞利用防范技术

**思路** ：针对**漏洞利用的条件进行干扰或拦截** ，防止攻击者成功利用漏洞

  1. 地址空间随机化技术（ASLR：通过对程序加载到内存的地址进行随机化处理，使攻击者在利用缓冲区溢出时不能事先确定程序返回值地址，降低攻击成功概率）
  2. 数据执行阻止（DEP：操作系统对特定的内存区域标注为非执行）
  3. SEHOP（Structured Exception Handler Overwrite Protection）
  4. 堆栈保护（Stack Protection：设置堆栈完整性标记以检测函数调用返回地址是否被篡改）
  5. 虚拟补丁（对尚未进行漏洞永久修复的目标系统程序，检测和过滤网络流量中的漏洞攻击数据包）

## 五、网络安全漏洞防护主要产品与技术指标

### 5-1 网络安全漏洞扫描器

  * 原理：利用已公开的漏洞信息及特征，通过程序对目标系统进行自动化分析，以确认目标系统是否存在响应的安全漏洞，简称“漏扫”
  * 国际：IBM Rational AppScan、Qualys、Shadow Security Scanner
  * 国内：绿盟远程安全评估系统、启明星辰天境脆弱性扫描与管理系统
  * 开源：Nessus、Nmap、OpenVAS
  * 技术指标：①漏洞扫描主机数量；②漏洞扫描并发数；③漏洞扫描速度；④漏洞检测能力（覆盖的数量、类型、OS、DB、网络设备）；⑤数据库漏洞检查能力；⑥Web 应用漏洞检查功能（SQL注入、XSS、网站木马、CGI漏洞）；⑦口令检查功能；⑧标准兼容性（CVE、CNNVD、CNVD、BugTraq）；⑧部署环境难易程度

### 5-2 网络安全漏洞服务平台

  * 漏洞盒子
  * 补天漏洞响应平台
  * 网络威胁情报服务

### 5-3 网络安全漏洞防护网关

  * 原理：通过从网络流量中提取和识别漏洞利用特征模式，阻止攻击者对目标系统的漏洞利用
  * 产品形式：IPS、WAF、UTM
  * 技术指标：①阻断安全漏洞攻击的种类与数量；②阻断安全漏洞攻击的准确率；③阻断安全漏洞攻击的性能；④支持网络带宽的能力
